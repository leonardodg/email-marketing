Funcionamento do Projeto

Requisitos:

  • 1: Git/Github para clonar repositório
  • 2: Ruby
  • 3: Node JS
  • 4: Compass
  • 5: Gulp e Plugins (npm install)
  • 6: Servidor Web

1 - Instalação Git:

O download para instalação do Git pode ser feito através do seguinte link:

https://git-scm.com/downloads

Escolha a opção adequada ao seu sistema e faça o download.

1.1 - Clonar repositório qisat/email-marketing

Abra o Git Bash, e acesse o diretório onde você deseja clonar o repositório, em seguida digite:

git clone git@github.com:qisat/email-marketing.git

Caso não consiga clonar o repositório pelo Git Bash, faça o download ZIP do repositório do projeto no link a seguir e extraia para a pasta que desejar:

https://github.com/qisat/email-marketing

2 - Instalação Ruby:

O download para instalação do Ruby pode ser feito através do seguinte link:

https://rubyinstaller.org/downloads/

Escolha a opção adequada ao seu sistema e faça o download.

3 - Instalação Node JS:

O download para instalação do Node JS pode ser feito através do seguinte link:

https://nodejs.org/en/download/

Escolha a opção adequada ao seu sistema e faça o download.

4 - Instalação Compass:

A instalação do Compass é feita de uma forma mais manual, será necessário acessar a linha de comando (CLI) e digitar os seguintes comandos respectivamente:

gem update --system

gem install compass

Copie e cole na linha de comando cada um dos comandos um de cada vez seguido da tecla Enter e aguarde as intalações e atualizações.

5 - Instalação Gulp:

  • Abra o prompt de comando (CLI)
  • Acesse o diretório raíz do projeto(diretório onde foi clonado o repositório email-marketing)
  • Digite "npm install", dê enter e aguarde...
  • Pronto, a instalação está feita.

6 - Servidor Web:

Caso já não possua um servidor web faça o download de um servidor do Chrome no link a seguir:

https://chrome.google.com/webstore/detail/web-server-for-chrome/ofhbbkphhbklhfoeikjpcbhemlocgigb?hl=en

7 - Editando Host

Abra a Linha de Comando (CLI) e digite o seguinte comando:

echo. && echo 127.0.0.1 teste.com.br >> %WINDIR%\System32\Drivers\Etc\Hosts

No trecho destacado deve ser digitado o endereço desejado para o host.

8 - Editando variáveis de destino no Gulp

No arquivo gulpfile.js, localizado na pasta raíz do projeto, deverá ser alterada a variável 'url_base'. O valor da variável deve ser igual ao endereço para o qual foi editado o host.

Sugestão:

Para editar o arquivo gulpfile.js é sugerido que se use um editor de texto. Caso você não tenha instale o Sublime Text 3 no link abaixo:

https://www.sublimetext.com/3

Gulp 4

Gulp 4

Caso você não tenha ainda o Gulp 4 e/ou os plugins necessários instalado na sua máquina, a instalação é simples, siga os passos abaixo:

  • Abra o prompt de comando (CLI)
  • Acesse o diretório raíz do projeto(diretório onde está o arquivo 'gulpfile.js' e o 'package.json') pelo prompt digitando "cd 'caminho até a pasta'" e digite enter;
  • Digite "npm install", dê enter e aguarde...
  • Pronto, a instalação está feita.

Detalhe:

O gulp deve sempre ser utilizado pelo CLI na pasta raíz do projeto onde deverá estar localizado o arquivo "gulpfile.js".

Funcionamento do Gulp para a produção de Campanhas

Para a Produção de uma campanha de Email será necessário a utilização do Gulp, portanto listamos a seguir as tarefas que serão realizadas por ele:

Automatização

O gulp como um automatizador de tarefas tem como função principal otimizar o tempo gasto com tarefas manuais, portanto criamos a tarefa "default", para evitar que todas as tarefas listadas sejam executadas uma a uma.

Tarefa default ("gulp")

Esta tarefa terá como função um "watch", que irá monitorar as pastas 'campanhas/2019/' 'public/images/icons/' e 'sass/foundation-scss/', assim havendo qualquer mudança em algumas das pastas, ele realizará todas as tarefas em sequência de forma automática.

A tarefa default pode ser chamada simplesmente digitando "gulp" na linha de comando no diretório raíz do projeto.

Tarefas do gulp a serem executadas

  • Tarefa 1: "gulp clean"
  • Tarefa 2: "gulp compass"
  • Tarefa 3: "gulp css-series"
  • Tarefa 4: "gulp replace-inline"
  • Tarefa 5: "gulp campanhas-inline"
  • Tarefa 6: "gulp campanhas-html-final"

Tarefa 1 - "gulp clean"

Esta tarefa será responsável por deletar o css antigo para que o gulp compile um novo com as mudanças após o ciclo do watch.
A tarefa clean só será executada no primeiro ciclo da tarefa default e não voltará a ser executada.

Tarefa 2 - "gulp compass"

Esta tarefa terá função de monitorar mudanças no sass e assim então compilar e criar os arquivos "qisat-foundation.css" e "qisat-foundation.min.css", que deverão estar localizados na pasta 'public/css'.

Tarefa 3 - "gulp css-series"

Esta tarefa terá a função de renomear os caminhos dos ícones para caminhos reais, e caminhos em relação a pasta raíz respectivamente.

Tarefa 4 - "gulp replace-inline"

Esta tarefa será responsável por alterar nos html os endereços raízes para endereços reais, para assim o css poder ser aplicado inline pela tarefa "campanhas-inline" posteiormente.

Tarefa 5 - "gulp campanhas-inline"

Esta tarefa seleciona todos os arquivos html do endereço "public/campanhas/2019/inline/", e insere inline o documento css externo utilizado nos html; Além de inserir estilos específicos no cabeçalho do email e também minificar o html.

Tarefa 6 - "gulp campanhas-html-final"

Esta tarefa é a última a ser realizada, ela minifica e transfere o html produzido em "campanhas/2019", para "public/campanhas/2019".

Importante:

Com a tarefa default sendo executada, ao renomear um arquivo na pasta 'campanhas/2019' o gulp criará um novo arquivo na pasta 'public/campanhas/2019' tendo então dois arquivos, portanto o arquivo de nome antigo deve ser deletado manualmente da pasta 'public/campanhas/2019'e da pasta 'public/campanhas/2019/inline'.

Variáveis

As variáveis "url-html", "url_imagens", "url_css" existem em função da variável "url_base" que é a única a ser modificada.
A variável url_base deverá ter como valor padrão "https://public.qisat.com.br", mas deve ser alterado conforme o host.
